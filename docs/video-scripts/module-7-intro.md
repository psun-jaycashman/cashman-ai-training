- **Target runtime:** ~80 seconds

- **Goal:** Define an "agent" in plain English, contrast it with a chatbot, and set up the key safety idea: capability is not the same as accuracy.

## Script

Peter again, and welcome to Module 7. This is where things get interesting.

Up to now, the AI we've used has worked like a chatbot. You type a question, it types an answer. Useful, but limited.

An **agent** is the next step. An agent doesn't just talk -- it can **take actions**.

Think of it this way. A chatbot is like talking to a really sharp intern who's read everything. An agent is the same intern, but now you've given them a desk, a computer, and access to your file systems. They can actually go look things up. Run a query. Pull a report. Stitch three steps together to get you a final answer.

In this module we'll cover three things.

First, **what agents can do.** I'll walk you through the difference between "ask a question" and "give the agent a goal."

Second, **using the agents that already live in the Cashman AI Portal.** We've built agents that can search documents, query project data, and aggregate numbers. You'll try one.

Third, **how to prompt an agent well.** Vague goals get vague results. Specific goals get powerful results. We'll practice the difference.

Now, here's the thing about agents you absolutely must remember: **capability is not the same as accuracy.** Agents look impressive. They search, they cite sources, they show their work. That can fool you into trusting them more than you should. An agent can confidently cite a document that doesn't actually say what the agent claims.

So treat an agent like a brilliant new hire on day one. Capable, helpful -- and not yet trusted to sign anything on your behalf. You stay in the loop. Let's get into it.
